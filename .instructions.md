# .instructions.md

## Project Name

Mahesh Perera - Software Engineering Portfolio

---

## Project Objective

Build a world-class personal portfolio website that positions Mahesh Perera as a high-potential Software Engineering Intern and Full-Stack Developer candidate.

The portfolio must showcase technical expertise, engineering projects, leadership experience, certifications, hackathon achievements, publications, and GitHub contributions.

The website should feel comparable to portfolios built by engineers hired at companies such as WSO2, 99x, Sysco Labs, IFS, Virtusa, Gapstars, and CodeGen.

---

## Core Principles

### 1. Recruiter First

The website is not a personal blog.

The website is a professional engineering portfolio.

Every section must help recruiters evaluate:

* Technical ability
* Software engineering mindset
* Problem-solving capability
* Leadership experience
* Project impact

---

### 2. Performance First

Requirements:

* Lighthouse Score 95+
* SEO Score 95+
* Accessibility Score 95+
* Mobile Friendly
* Responsive Design
* Fast Page Load

---

### 3. Engineering Focus

The website must highlight:

* Software Engineering
* Full-Stack Development
* Cloud Engineering
* AI Applications
* Mobile Development

Avoid unnecessary personal content.

---

### 4. Modern Development Standards

Use:

* Next.js 15
* TypeScript
* Tailwind CSS
* ShadCN UI
* Framer Motion
* Lucide Icons

---

### 5. Clean Architecture

Organize code using:

/app
/components
/data
/lib
/hooks
/types
/public

Avoid monolithic components.

Reusable components only.
